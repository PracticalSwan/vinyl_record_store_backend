# Backend Project Context

This is the backend source of truth for the Vinyl Record Store Recommender System.

## Backend Role

The backend supports the vinyl record store by providing data, APIs, recommendation services, and interaction logging for the separate frontend.

The backend does not own customer-facing React screens. It owns the server behavior that lets the frontend browse records, log actions, and request recommendations.

## Academic Focus

The academic focus is Decision Support and Recommender Systems. Backend work should make recommendation ranking, explanations, and evaluation evidence clear.

## Scope

Backend scope:

- Product catalog APIs.
- Product detail API.
- Search and filter API.
- User interaction logging.
- Wishlist, cart, and order-related API behavior.
- MongoDB Atlas data access.
- Content-based recommender service for the MVP.
- Recommendation logs for evaluation.
- Server-side validation and error handling.

## Non-Goals

These are out of scope unless the user explicitly approves them:

- Frontend UI implementation.
- Production payment processing.
- Production authentication.
- Scraping external websites.
- Copying product data from external websites.
- Collaborative filtering before enough interaction data exists.
- Real database connection code during planning tasks.

## Assumptions

- The backend uses Next.js.
- The frontend lives in `../vinyl_record_store_frontend`.
- MongoDB Atlas stores products, users, interactions, orders, wishlists, and recommendation logs.
- API responses should be easy for the frontend to consume.
- Recommendation explanations should be generated by backend logic and displayed by the frontend.
- No real MongoDB credentials are available in the repository.

## Constraints

- Current phase is planning/setup only.
- Keep secrets out of source control.
- Keep backend logic separate from frontend UI.
- Keep docs editable.
- Keep explanations beginner-friendly.

## Important Terms

- API contract: the agreed request and response shape between frontend and backend.
- Interaction: a user action such as view, wishlist, cart add, purchase, rating, like, dislike, or search.
- Content-based recommendation: a method that recommends records using metadata similarity.
- Recommendation log: stored output from a recommendation request for debugging and evaluation.
- Algorithm version: a label that helps compare recommender changes over time.

## Documentation Maintenance Rule

Update backend docs when backend behavior, setup, architecture, API contracts, data models, recommender logic, validation, environment variables, risks, or scope change.

If information is missing, add a clear `TODO` with the missing decision.

## Backend Task Workflow

1. Read `LESSONS.md`, `AGENTS.md`, `CLAUDE.md`, and relevant docs.
2. Identify the backend task type.
3. Make a short plan for multi-file work.
4. Modify only relevant backend files.
5. Validate when practical.
6. Update affected backend docs.
7. Check whether frontend integration docs need updates.
8. Summarize changed files, validation, assumptions, and next steps.
